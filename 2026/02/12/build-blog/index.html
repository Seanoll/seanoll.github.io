  1. 环境准备
    • 安装Node.js
    • 安装Git
    • 初始化博客
  2. 部署到Github Pages
    • 创建Github仓库，名字为：你的Github用户名.github.io
    • 配置Hexo部署信息，打开本地博客根目录下的 _config.yml文件，找到最底部的 deploy字段
    • 安装部署插件
    • 回到Github仓库页面，点击Settings -> Pages， 在 Source下拉菜单中，选择 Deploy from a branch，然后选择刚才配置的分支（如 main或 master），点击 Save
  3. 内容
    • 在Hexo官网或者Github上找主题，下载到themes文件夹或者git clone
    • 在博客根目录下的_config.yml中下拉找到theme改为theme: 什么名字
    • 在主题的source文件夹里修改配置文件
    • 使用hexo new 文章名来新建一篇文章，找到并编辑它，写好之后保存
    • 使用hexo new page 网页名来新建一张页面
    • 使用hexo g来生成相应的文件，使用hexo d部署更新到网站上
  4. 在配置过程中出现版本问题
    • 在命令行里无法删除一个文件夹Permission Denied可能是因为该文件夹在图形化界面中打开。
    • hexo init必须初始化一个空文件夹，如果已经删干净了还是提示有文件，使用ls -a来查看隐藏文件并删除。
    • git clone或者wget和curl速度太慢时，将网络代理设置成全局模式。
    • 在 YAML 中，冒号 :后面必须有一个空格。branch:master应该写为 branch: master。非末尾的后大括号后面需要加逗号,。
